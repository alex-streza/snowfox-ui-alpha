# Customization

When you load a **snowfox** component and you've previously loaded the corresponding `snowfox.css` file, a basic style library will be added to the component.
To easy customize the different components that this library brings, we have provided two different methods, using **BEM Classes** and **CSS Custom Properties**.

## BEM Classes

We are always exporting CSS classes in our components and we use [BEM to decide the names](http://getbem.com) for each part of the component.

Example of BEM classes:

```html
<div className="sf-author sf-author--primary sf-author--rounded">
  <span className="sf-author__name">Alicia</span>
  <span className="sf-author__subtitle">Spain</span>
</div>
```

We are also using other CSS conventions like [States from SMACSS](http://smacss.com/book/type-state), to make the component more readable.

Example of SMACSS States:

```html
<div className="sf-author is-disabled">...</div>
```

The previous convention plus **Reponsive Suffixes** are part of **BEMIT** and we use them to make the component more responsive, those where introduced in [CSS Wizardy blog post](https://csswizardry.com/2015/08/bemit-taking-the-bem-naming-convention-a-step-further/) and an example of this used could be:

```html
<div className="sf-author is-hidden@xs">...</div>
```

Our configured **Responsive Suffixes** are the same as our default breakpoints:

```css
--sf-breakpoint-xxs: 360px; // @xxs
--sf-breakpoint-xs: 640px; // @xs
--sf-breakpoint-s: 768px; // @s
--sf-breakpoint-m: 1024px; // @m
--sf-breakpoint-l: 1280px; // @l
--sf-breakpoint-xl: 1440px; // @xl
--sf-breakpoint-xxl: 1920px; // @xxs
```

## CSS Custom Properties

We have defined a list of custom properties that we use to customize the components. You can easily override them in your project.

For example if we want to modify the **variants**, there are 3 different groups of properties:

```css
/* Color Primary */
--sf-color-primary-brightest: #b9daff;
--sf-color-primary-brighter: #a4cfff;
--sf-color-primary-bright: #53a5ff;
--sf-color-primary-normal: #017aff;
--sf-color-primary-dark: #005fc8;
--sf-color-primary-darker: #004590;
--sf-color-primary-darkest: #002a58;

/* Color Secondary */
--sf-color-secondary-brightest: #eeeaff;
--sf-color-secondary-brighter: #cec1ff;
--sf-color-secondary-bright: #ad99ff;
--sf-color-secondary-normal: #6c47ff;
--sf-color-secondary-dark: #3f0fff;
--sf-color-secondary-darker: #2b00d6;
--sf-color-secondary-darkest: #20009e;

/* Color Tertiary */
--sf-color-tertiary-brightest: #fde2e8;
--sf-color-tertiary-brighter: #fbbccb;
--sf-color-tertiary-bright: #f896ad;
--sf-color-tertiary-normal: #f34971;
--sf-color-tertiary-dark: #f01448;
--sf-color-tertiary-darker: #bf0d37;
--sf-color-tertiary-darkest: #8b0928;
```

We have also defined some **intents** to be used in some components:
